# Настройка Bitrix24

Интеграция работает через сервер `server.js`, поэтому входящий вебхук Bitrix24 не попадает в браузер и не виден посетителям сайта.

## 1. Создать входящий вебхук

В Bitrix24 откройте:

`Приложения` -> `Ресурсы разработчика` -> `Готовые сценарии` -> `Другое` -> `Входящий вебхук`

Права вебхука: `CRM`.

Скопируйте URL вида:

```env
https://your-domain.bitrix24.ru/rest/1/xxxxxxxxxxxxxxx/
```

## 2. Заполнить `.env`

```env
BITRIX_WEBHOOK=https://your-domain.bitrix24.ru/rest/1/xxxxxxxxxxxxxxx/
BITRIX_STAGE_ID=NEW
BITRIX_SOURCE_ID=WEB
BITRIX_ASSIGNED_BY_ID=
BITRIX_TIMEOUT_MS=10000
```

`BITRIX_ASSIGNED_BY_ID` можно оставить пустым. Если нужен конкретный ответственный менеджер, укажите его ID из Bitrix24.

## 3. Проверить

Запустите сайт:

```bash
npm start
```

Проверка настроек:

```bash
http://127.0.0.1:3000/api/bitrix-status
```

Тестовая сделка:

```bash
http://127.0.0.1:3000/api/test-bitrix
```

## Что отправляется в Bitrix24

- форма обратного звонка создает контакт и сделку;
- заказ из калькулятора создает контакт и сделку;
- телефон используется для поиска существующего контакта;
- если контакта нет, он создается;
- сумма заказа попадает в `OPPORTUNITY`;
- детали расчета попадают в комментарий сделки.
